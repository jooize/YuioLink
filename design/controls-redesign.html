Expiry · Limit · Privacy — control layouts

Four ways to set the three knobs on the create form. Key realisation from the namespace work: Limit and Privacy collapse into one choice, because the valid combinations are only three — a short guessable name can't be a one-time link (a guesser would burn it), so "short + one-time" doesn't exist. That leaves a clean 1-D scale.

The three real options (each also picks an expiry, 1 min – 7 days):
• Public — reusable · 1 word (yuio.link/mango) · short, guessable. Convenient, not private.
• Private — reusable · 4 words (runnyDUSKoutletDELUGE) · unguessable. Only those you send it to can find it.
• One-time — single-use · 4 words · unguessable, opens once.

Recommendations

  1. Ship #1 (two rows + live preview). Smallest change from today, keyboard/no-JS friendly, and the live name+verdict turns the entropy story into something you can see. The "Type" row replaces the old "Limit views to".
  2. Steal the preview strip for any variant. Seeing mango vs runnyDUSK… is what makes Public-vs-Private click.
  3. #4 (presets) is the best "first 5 seconds" UX if we want to optimise for newcomers — pair it with #1 behind "Advanced". Good later, more build now.
  4. #2 (sentence) is the friendliest but hides custom expiry; nice for a marketing-y landing, harder to keep accessible.
  5. Either way: rename the legend from "Limit views to" → "Type" (or "Privacy"), and drop the standalone ∞/Once control.
